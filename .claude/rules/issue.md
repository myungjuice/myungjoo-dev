# 이슈 컨벤션

## 이슈 타입

| 타입       | 용도                           |
| ---------- | ------------------------------ |
| `feat`     | 새 기능 추가                   |
| `fix`      | 버그 수정                      |
| `refactor` | 코드 리팩토링 (기능 변경 없음) |
| `chore`    | 설정, 패키지, 기타 작업        |
| `docs`     | 문서 작성/수정                 |
| `style`    | UI/스타일 변경                 |
| `perf`     | 성능 개선                      |

## 제목 형식

```
[feat] 기능명
[fix] 버그명
```

## 본문 구성

이슈는 기획 관점에서 작성한다.

- **무엇을 만들 것인지**: 기능/목적
- **왜 필요한지**: 이유/배경
- **어떤 결과를 기대하는지**: 완료 조건

> 개발 구현 내역, 변경 파일 목록 등 기술적 세부사항은 PR에 작성한다.

## 타입 한글 표기

제목은 영문 타입을 사용하고, 본문에서 타입을 언급할 때는 한글로 작성한다.

| 영문     | 한글      |
| -------- | --------- |
| feat     | 기능 추가 |
| fix      | 버그 수정 |
| refactor | 리팩토링  |
| chore    | 기타 작업 |
| docs     | 문서      |
| style    | 스타일    |
| perf     | 성능 개선 |

## 규칙

- 이슈 작성 및 답변은 한글로 작성
